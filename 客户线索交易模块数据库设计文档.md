## 一、数据库概述

### 1.1 设计原则
- **第三范式(3NF)**: 减少数据冗余,保证数据一致性
- **性能优化**: 合理使用索引,避免全表扫描
- **安全性**: 敏感数据加密存储
- **可扩展性**: 预留扩展字段,支持业务增长

### 1.2 命名规范
- **表名**: 小写字母+下划线,复数形式,如 `users`, `leads`
- **字段名**: 小写字母+下划线,如 `user_id`, `created_at`
- **索引名**: `idx_表名_字段名` 或 `uk_表名_字段名`(唯一索引)
- **外键名**: `fk_表名_关联表名`

### 1.3 数据类型选择
- **主键**: `BIGINT UNSIGNED` (自增)
- **状态字段**: `TINYINT UNSIGNED` (0-255)
- **金额字段**: `INT` (单位:分) 或 `DECIMAL(10,2)`
- **文本字段**: `VARCHAR` (长度≤255) 或 `TEXT` (长度>255)
- **时间字段**: `DATETIME` 或 `TIMESTAMP`

### 1.4 加密策略
- **手机号**: AES-256加密
- **身份证**: AES-256加密
- **客户姓名**: 可选加密(脱敏展示)
- **敏感操作日志**: 记录操作前后的关键字段

## 二、核心表结构设计

### 2.1 用户表 (users)

**功能**: 存储用户基本信息和认证状态

```sql
CREATE TABLE `users` (
  `user_id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `phone` VARCHAR(255) NOT NULL COMMENT '手机号(加密存储)',
  `real_name` VARCHAR(100) NOT NULL COMMENT '真实姓名',
  `id_card` VARCHAR(255) NOT NULL COMMENT '身份证号(加密存储)',
  `email` VARCHAR(100) DEFAULT NULL COMMENT '邮箱',
  `avatar` VARCHAR(255) DEFAULT NULL COMMENT '头像URL',
  `auth_status` TINYINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '认证状态: 0-未认证,1-已认证,2-高级认证',
  `face_verified` TINYINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '是否人脸识别: 0-否,1-是',
  `credit_score` INT NOT NULL DEFAULT 100 COMMENT '信用分(0-100)',
  `credit_level` VARCHAR(10) NOT NULL DEFAULT 'B' COMMENT '信用等级: S/A/B/C/D',
  `last_login_ip` VARCHAR(50) DEFAULT NULL COMMENT '最后登录IP',
  `last_login_time` DATETIME DEFAULT NULL COMMENT '最后登录时间',
  `is_blocked` TINYINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '是否被封禁: 0-否,1-是',
  `blocked_reason` VARCHAR(255) DEFAULT NULL COMMENT '封禁原因',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '注册时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

  PRIMARY KEY (`user_id`),
  UNIQUE KEY `uk_phone` (`phone`(100)) COMMENT '手机号唯一索引(前缀索引)',
  KEY `idx_auth_status` (`auth_status`) COMMENT '认证状态索引',
  KEY `idx_credit_score` (`credit_score`) COMMENT '信用分索引',
  KEY `idx_created_at` (`created_at`) COMMENT '注册时间索引'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';

-- 认证协议签署记录表
CREATE TABLE `user_agreements` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT UNSIGNED NOT NULL COMMENT '用户ID',
  `agreement_type` VARCHAR(50) NOT NULL COMMENT '协议类型: compliance-合规承诺书, gig-众包协议',
  `agreement_version` VARCHAR(20) NOT NULL COMMENT '协议版本号',
  `signed_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '签署时间',
  `ip_address` VARCHAR(50) DEFAULT NULL COMMENT '签署IP',

  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_agreement_type` (`agreement_type`),
  CONSTRAINT `fk_user_agreements_user` FOREIGN KEY (`user_id`) REFERENCES `users` (`user_id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户协议签署记录表';
```

**字段说明**:

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| user_id | BIGINT | 是 | 自增 | 主键 |
| phone | VARCHAR(255) | 是 | - | 手机号,AES加密后存储 |
| real_name | VARCHAR(100) | 是 | - | 真实姓名,明文存储 |
| id_card | VARCHAR(255) | 是 | - | 身份证号,加密存储 |
| auth_status | TINYINT | 是 | 0 | 0=未认证,1=已认证,2=高级认证 |
| face_verified | TINYINT | 是 | 0 | 0=未人脸认证,1=已人脸认证 |
| credit_score | INT | 是 | 100 | 信用分,初始100分 |
| credit_level | VARCHAR(10) | 是 | 'B' | S(90-100)/A(80-89)/B(70-79)/C(60-69)/D(<60) |

**索引说明**:
- `uk_phone`: 手机号唯一索引,使用前缀索引(100字符)避免加密后长度过长
- `idx_auth_status`: 认证状态索引,用于筛选已认证用户
- `idx_credit_score`: 信用分索引,用于信用等级排序

### 2.2 用户账户表 (accounts)

**功能**: 存储用户的积分余额信息

```sql
CREATE TABLE `accounts` (
  `account_id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '账户ID',
  `user_id` BIGINT UNSIGNED NOT NULL COMMENT '用户ID',
  `recharge_balance` INT NOT NULL DEFAULT 0 COMMENT '充值分余额(分)',
  `income_balance` INT NOT NULL DEFAULT 0 COMMENT '收益分余额(分)',
  `frozen_balance` INT NOT NULL DEFAULT 0 COMMENT '冻结余额(分)',
  `total_recharge` INT NOT NULL DEFAULT 0 COMMENT '累计充值(分)',
  `total_income` INT NOT NULL DEFAULT 0 COMMENT '累计收益(分)',
  `total_withdraw` INT NOT NULL DEFAULT 0 COMMENT '累计提现(分)',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

  PRIMARY KEY (`account_id`),
  UNIQUE KEY `uk_user_id` (`user_id`) COMMENT '用户唯一索引',
  KEY `idx_recharge_balance` (`recharge_balance`) COMMENT '充值余额索引',
  KEY `idx_income_balance` (`income_balance`) COMMENT '收益余额索引',

  CONSTRAINT `fk_accounts_user` FOREIGN KEY (`user_id`) REFERENCES `users` (`user_id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户账户表';

-- 账户流水表
CREATE TABLE `account_transactions` (
  `transaction_id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '交易流水ID',
  `user_id` BIGINT UNSIGNED NOT NULL COMMENT '用户ID',
  `account_id` BIGINT UNSIGNED NOT NULL COMMENT '账户ID',
  `transaction_type` VARCHAR(50) NOT NULL COMMENT '交易类型: recharge-充值, purchase-购买, sale-卖出, refund-退款, withdraw-提现, withdraw_cancel-提现取消',
  `amount` INT NOT NULL COMMENT '交易金额(分,可负数)',
  `balance_before` INT NOT NULL COMMENT '交易前余额(分)',
  `balance_after` INT NOT NULL COMMENT '交易后余额(分)',
  `related_id` VARCHAR(100) DEFAULT NULL COMMENT '关联ID: 订单ID/提现ID等',
  `description` VARCHAR(255) DEFAULT NULL COMMENT '交易描述',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',

  PRIMARY KEY (`transaction_id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_transaction_type` (`transaction_type`),
  KEY `idx_related_id` (`related_id`),
  KEY `idx_created_at` (`created_at`),

  CONSTRAINT `fk_account_transactions_account` FOREIGN KEY (`account_id`) REFERENCES `accounts` (`account_id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='账户交易流水表';
```

**字段说明**:

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| account_id | BIGINT | 是 | 自增 | 主键 |
| user_id | BIGINT | 是 | - | 用户ID,外键 |
| recharge_balance | INT | 是 | 0 | 充值分余额,仅用于购买 |
| income_balance | INT | 是 | 0 | 收益分余额,可提现 |
| frozen_balance | INT | 是 | 0 | 冻结余额,售后保护期内 |
| total_recharge | INT | 是 | 0 | 累计充值金额(历史统计) |
| total_income | INT | 是 | 0 | 累计收益金额(历史统计) |

**业务规则**:
- 充值分只能用于购买线索,不可提现
- 收益分来自卖出线索,可申请提现
- 冻结余额在售后保护期内不可操作,24小时后自动转为收益分

```sql
  `company` VARCHAR(200) DEFAULT NULL COMMENT '公司/单位',

  `ai_recommend_max` INT DEFAULT NULL COMMENT 'AI推荐最高价(分)',
  `status` TINYINT UNSIGNED NOT NULL DEFAULT 0 COMMENT '状态: 0-待审核,1-已上架,2-已售出,3-已下架,4-问题线索',

  `verify_result` VARCHAR(255) DEFAULT NULL COMMENT '校验结果说明',

```

**字段说明**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| lead_id | BIGINT | 是 | 主键 |
| seller_id | BIGINT | 是 | 卖家用户ID |
| customer_phone_hash | VARCHAR(64) | 是 | 手机号的SHA256哈希,用于48小时内去重 |

- 线索编号格式: `LD + YYYYMMDD + 6位序号`,如 `LD20260311000001`

- 软删除机制,保留数据用于统计和审计

### 2.4 订单表 (orders)

**功能**: 存储交易订单信息

**字段说明**:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| order_id | VARCHAR(50) | 是 | 主键,格式: ORD+YYYYMMDD+6位序号 |
| status | TINYINT | 是 | 0=待支付,1=已支付,2=已退款,3=已完成 |
| refund_status | TINYINT | 是 | 0=无退款,1=退款中,2=已退款,3=部分退款 |
| after_sale_deadline | DATETIME | 是 | 购买时间+24小时,用于售后保护期判断 |

**状态流转规则**:

```
待支付(0)
    ↓ 支付成功
已支付(1) ←──────────────┐
    ↓ 24小时内无申诉      │
已完成(3)                │
    ↑                    │
已退款(2) ← 24小时内申诉成立 │
    或 部分退款(3+部分退款标识)
```

## 四、索引优化策略

### 4.1 单表索引优化

#### users表
- `uk_phone`: 唯一索引,登录和认证使用
- `idx_credit_score`: 排序和筛选使用
- 考虑添加复合索引 `idx_auth_credit` (auth_status, credit_score) 用于优质卖家筛选

#### leads表
- `idx_city_intent`: 联合索引,用于地域和意向筛选
- `idx_status_price`: 联合索引 (status, price) 用于展示已上架线索并按价格排序
- `idx_seller_status`: 联合索引 (seller_id, status) 用于卖家查看自己的线索

#### orders表
- `idx_buyer_status`: 联合索引 (buyer_id, status) 用于买家查看订单
- `idx_seller_status`: 联合索引 (seller_id, status) 用于卖家查看订单
- `idx_after_sale_deadline`: 用于定时任务扫描

#### complaints表
- `idx_order_status`: 联合索引 (order_id, status) 用于查询订单申诉状态
- `idx_seller_status`: 联合索引 (seller_id, status) 用于卖家查看被申诉记录

### 4.2 慢查询优化

#### 线索列表查询优化
```sql
-- 优化前: 全表扫描
SELECT * FROM leads WHERE status = 1 ORDER BY created_at DESC LIMIT 20;

-- 优化后: 使用联合索引
CREATE INDEX idx_status_created ON leads(status, created_at);
SELECT * FROM leads WHERE status = 1 ORDER BY created_at DESC LIMIT 20;
```

#### 订单统计查询优化
```sql
-- 优化前: 多表JOIN,慢
SELECT COUNT(*), SUM(amount) FROM orders
WHERE buyer_id = 123 AND status = 1 AND DATE(created_at) = CURDATE();

-- 优化后: 添加复合索引
CREATE INDEX idx_buyer_status_date ON orders(buyer_id, status, created_at);
```

### 4.3 定期维护任务


### 8.2 SQL示例

#### 创建线索编号
```sql
-- 存储过程: 生成线索编号
DELIMITER $$
CREATE PROCEDURE generate_lead_no(OUT lead_no VARCHAR(50))
BEGIN
    DECLARE today_date VARCHAR(8);
    DECLARE seq_num INT;

    SET today_date = DATE_FORMAT(NOW(), '%Y%m%d');

    SELECT COUNT(*) + 1 INTO seq_num
    FROM leads
    WHERE lead_no LIKE CONCAT('LD', today_date, '%');

    SET lead_no = CONCAT('LD', today_date, LPAD(seq_num, 6, '0'));
END$$
DELIMITER ;

-- 调用
CALL generate_lead_no(@new_lead_no);
SELECT @new_lead_no;
```

#### 信用分计算
```sql
-- 更新卖家信用分(申诉成立时)
UPDATE users u
INNER JOIN complaints c ON u.user_id = c.seller_id
SET u.credit_score = GREATEST(0, u.credit_score - 10),
    u.credit_level = CASE
        WHEN u.credit_score - 10 >= 90 THEN 'S'
        WHEN u.credit_score - 10 >= 80 THEN 'A'
        WHEN u.credit_score - 10 >= 70 THEN 'B'
        WHEN u.credit_score - 10 >= 60 THEN 'C'
        ELSE 'D'
    END
WHERE c.complaint_id = 123
  AND c.status = 2;  -- 申诉已通过
```
