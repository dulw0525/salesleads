## 一、文档概述

### 1.1 产品定位
聚流 App 是一款面向保险代理人的客户经营互助平台，通过合规的"资源分享"机制实现线索流转，并与私域客户中心（CRM）深度联动。

### 1.2 核心设计原则
- **行为去商业化**：以积分体系替代直接金钱交易
- **资金劳务化**：提现定义为"推广劳务报酬"
- **系统自治化**：自动化仲裁与信用约束机制
- **合规金融架构**：双轨制积分确保资金进出的法律合规性
- **闭环生态**：线索申领后自动同步至客户中心，形成完整经营闭环

### 1.3 业务流程概览
客户线索流转遵循 **"分享 → 大厅 → 申领"** 三段式业务流，确保信息真实性与经营效率。

## 二、核心业务流程

### 2.1 客户线索分享 (Share)

#### 2.1.1 角色定义
- **分享人**：拥有客户线索资源但无法及时跟进或跨地域的保险代理人
- **目标用户**：需要将手头客户资源流转变现的代理人

#### 2.1.2 功能流程
```
开始 → 录入客户画像 → 签署承诺函 → 预检机制 → 提交成功
```

#### 2.1.3 详细功能规格

**（1）客户画像信息采集**
- **必填字段**：
  - 客户年龄段（如：25-30岁、30-35岁等）
  - 性别
  - 职业
  - 意向险种（下拉选择：寿险、健康险、意外险、教育金、养老险等）
  - 保费预算区间（如：1-5万、5-10万、10-20万等）
  - 所在地域（省份/城市）
  - 客户意向程度（高/中/低）

- **选填字段**：
  - 购买时间预期（1个月内/3个月内/半年内/1年以上）
  - 客户购买习惯（线上偏好/线下偏好/无明显偏好）
  - 备注信息（特殊需求说明）

**（2）合规背书机制**
- **强制签署**：《信息来源合法及已获授权承诺函》
- **签署内容**：
  - 确认客户信息来源合法
  - 确认已获得客户授权用于保险咨询
  - 承诺不侵犯客户隐私权
  - 违约承担相应法律责任
- **签署方式**：电子签名 + 确认框勾选
- **法律效力**：系统记录签署时间与IP，具备法律追溯能力

**（3）预检机制**
- **空号/实名校验**：提交后后端调用 API 进行号码活性校验
- **校验结果**：
  - 校验通过：进入正常分享流程
  - 校验失败：提示用户重新录入或放弃分享

**（4）激励积分发放**
- **积分获取**：预检通过后，系统向分享人发放"待生效"状态的**激励积分**
- **积分数值**：50 积分（固定值）
- **积分状态**：标记为"冻结"状态，待申领人反馈后转为"可用"

**（5）界面交互流程**
1. 点击"分享线索"按钮进入分享页
2. 填写客户画像表单（带校验提示）
3. 勾选承诺函（未勾选无法提交）
4. 提交分享
5. 系统调用预检机制
6. 成功提示："分享成功，已获得50激励积分（待生效）"

### 2.2 客户线索大厅 (Marketplace)

#### 2.2.1 角色定义
- **浏览者**：对大厅画像感兴趣的意向申领人
- **目标用户**：需要获取新客户资源的保险代理人

#### 2.2.2 大厅界面设计

**（1）筛选维度**
- **险种类型**：寿险、健康险、意外险、教育金、养老险等
- **保费预算**：1-5万、5-10万、10-20万、20万以上
- **地域范围**：全国、省份、城市
- **更新时间**：今日、3日内、7日内

**（2）列表展示字段**
- 客户画像卡片包含：
  - 年龄段标签
  - 职业标签
  - 意向险种标签
  - 保费预算标签
  - 地域标签
  - 客户意向程度（高/中/低）
  - 分享人信誉分（展示星级或分数）
  - 发布时间（如：2小时前、1天前）
  - 有效时间倒计时（剩余天数）

**（3）脱敏浏览**
- 大厅仅展示客户画像标签、有效时间及分享人的**信誉分**
- 不展示客户真实姓名、电话号码等敏感信息

**（4）视觉规范**
- 采用沉浸式卡片设计
- 申领按钮高亮显示为品牌绿（#00bc71）
- 延续 AI 科技风格，以黑色为底色

#### 2.2.3 线索详情页面功能

**（1）线索详情展示**
- 点击线索卡片进入线索详情页面
- 详情页面展示完整客户画像信息：
  - 客户年龄段、性别、职业
  - 意向险种、保费预算
  - 所在地域、客户意向程度
  - 分享人信息（信誉分）
  - 发布时间、有效时间

**（2）申领入口**
- 详情页面底部显示"申领线索"按钮
- 点击按钮后直接进入线索申领流程

**（3）申领流程**
```
浏览线索 → 点击线索卡片 → 查看详情页面 → 点击"申领线索"
→ 系统校验积分余额 → 显示申领确认框 → 确认申领
→ 扣除积分 → 申领成功 → 同步至客户中心
```

**（4）界面交互设计**
- 详情页面采用卡片式布局，信息清晰易读
- 支持上下滑动查看更多信息
- "申领线索"按钮固定在页面底部，方便操作
- 申领前提示："申领该线索需消耗积分，确认申领？"

### 2.3 客户线索申领 (Claim & Sync)

#### 2.3.1 角色定义
- **申领人**：浏览线索详情后，决定正式经营该客户的用户
- **目标用户**：准备跟进的代理人

#### 2.3.2 申领权限获取

**（1）积分消耗规则**
- **申领费用**：消耗"聚流积分"获取经营权
- **费用计算**：系统根据线索价值计算消耗积分数量
- **积分不足处理**：
  - 余额不足时提示："积分不足，请先充值或完成其他任务"
  - 提供充值入口和积分任务入口

**（2）申领操作流程**
1. 在线索详情页面点击"申领线索"按钮
2. 系统校验积分余额
3. 显示申领确认框（含费用说明和经营保护条款）
4. 点击确认，扣除积分
5. 申领成功，进入 CRM 客户中心

#### 2.3.3 数据联动（CRM）

**（1）自动同步**
- 申领成功后，系统自动将客户画像同步至申领人的**【客户中心】**
- 同步字段：年龄段、职业、意向险种、保费预算、地域、意向程度

**（2）来源标记**
- 客户来源字段自动锁定为**"线索引擎"**
- 不可修改，确保数据溯源

**（3）盘活追踪**
- 一旦申领人发起有效跟进，客户将自动进入**【盘活客户列表】**
- 有效跟进定义：拨打电话、添加备注、更新状态等

**（4）客户中心功能**
- 可查看客户完整画像
- 记录跟进进度
- 添加客户标签
- 设置提醒事项

#### 2.3.4 隐私通讯

**（1）虚拟号系统**
- **拨号方式**：通过内置**虚拟号（隐私拨号）**联系客户
- **双向匿名**：保护双方隐私，数据不脱离平台
- **虚拟号生成**：系统自动生成中间号码
- **通话记录**：系统自动保存通话时长、拨打次数

**（2）拨号权限**
- **申领成功**：获得客户姓氏（如："王先生"、"李女士"）
- **隐私拨号**：系统生成虚拟号码，可直接拨打
- **有效期**：虚拟号与申领保护期绑定

#### 2.3.5 反馈闭环机制

**（1）反馈要求**
- **反馈时限**：申领成功后48小时内必须反馈
- **反馈选项**：
  - 真实有效：客户信息属实，已建立联系
  - 虚假信息：客户不存在或需求不真实
  - 无法联系：拨打多次未接通
  - 其他原因：填写具体说明

**（2）反馈触发的系统动作**

**情况A：反馈"真实有效"**
- 分享人账户的激励积分由"待生效"转为"可用"
- 申领人获得积分奖励（鼓励反馈）
- 线索状态变更为"已完成"

**情况B：反馈"虚假信息"**
- 启动自动仲裁流程（见4.1节）
- 暂停分享人该线索的积分发放
- 记录投诉，进入审核队列

**情况C：反馈"无法联系"**
- 分享人积分继续待生效，等待二次确认
- 系统自动标记该线索为"联系困难"
- 给予二次反馈期

**情况D：超时未反馈**
- 系统自动发送提醒
- 超时后自动标记为"反馈超时"
- 分享人积分继续待生效，等待仲裁

## 三、聚流积分体系：双轨制与分类管理

系统严格区分积分属性，确保资金进出的法律合规性。

### 3.1 积分类型定义

| 积分属性 | 获取来源 | 提现权限 | 核心逻辑 |
| :---- | :---- | :---- | :---- |
| **充值积分** | 人民币购买"展业能量包" | **禁止提现** | 属于平台技术服务费，仅限站内消耗 |
| **激励积分** | 分享真实线索、平台任务 | **支持提现** | 属于平台支付给用户的**劳务推广报酬** |

#### 3.1.1 充值积分
- **来源**：用户购买"展业能量包"（如：10元=100积分）
- **标记属性**：不可提现、仅用于消费
- **使用场景**：申领线索、购买增值服务
- **过期规则**：无有效期限制

#### 3.1.2 激励积分
- **来源1**：分享真实线索且被评价为真实后获得
- **来源2**：邀请新用户注册并完成首单
- **来源3**：参与平台活动获得奖励
- **标记属性**：支持提现（需满足条件）
- **使用场景**：所有消费场景 + 提现
- **过期规则**：12个月未使用自动过期

### 3.2 积分账户管理

#### 3.2.1 账户结构
```
总积分余额 = 充值积分 + 激励积分
可用积分余额 = 总余额 - 冻结积分
提现可用余额 = 激励积分余额 - 冻结激励积分
```

#### 3.2.2 冻结机制
- **分享冻结**：分享线索后，预分配的激励积分进入冻结账户
- **仲裁冻结**：争议线索的积分暂时冻结，等待仲裁结果
- **提现冻结**：提现申请提交后，对应积分冻结直至到账

#### 3.2.3 积分明细记录
- 每笔积分变动生成流水记录
- 记录字段：时间、类型、来源/去向、数量、余额、备注
- 用户可随时查看积分明细

### 3.3 独立提现校验机制

#### 3.3.1 来源校验
- **提取限制**：提现时系统仅调取"激励积分"余额
- **充值积分**：无法被选中用于提现
- **自动过滤**：系统自动区分积分类型

#### 3.3.2 合规流程
- **劳务协议**：提现需签署《劳务合作协议》
- **个税代扣**：平台代扣代缴个税
- **结算凭证**：生成《劳务报酬结算凭证》

#### 3.3.3 身份一致性校验
- **实名认证**：用户必须完成实名认证（身份证+人脸识别）
- **银行卡绑定**：提现银行卡必须与实名信息一致
- **三位一体校验**：
  ```
  提现银行卡姓名 == 实名认证姓名 == App注册主体
  三者必须完全一致，否则提现失败
  ```

#### 3.3.4 信用硬约束
- **信誉分要求**：信誉分 ≥ 90分（满分100分）
- **投诉检查**：无未处理的投诉或仲裁案件
- **违规记录**：近30天无严重违规记录

#### 3.3.5 提现流程
```
1. 用户点击"提现" → 2. 系统仅显示可用激励积分
3. 填写提现金额 → 4. 系统校验身份与信用
5. 确认提现信息 → 6. 提交提现申请
7. 系统冻结对应积分 → 8. 生成提现订单
9. 平台审核（自动+人工） → 10. 财务打款
11. 用户确认到账 → 12. 订单完成
```

**提现时间**：提交后3-5个工作日到账

### 3.4 税务合规处理

#### 3.4.1 账单定义
- **充值账单**：定义为"技术服务费"
- **提现账单**：定义为"推广劳务报酬"

#### 3.4.2 个税代扣代缴
- **起征点**：单次提现金额 > 800元
- **税率**：按劳务报酬税率（20%、30%、40%累进）
- **自动计算**：系统自动计算应扣税额
- **显示明细**：
  ```
  申请提现：1000元
  个税扣除：200元
  实际到账：800元
  ```

#### 3.4.3 凭证生成
- **自动生成**：《劳务报酬结算凭证》
- **凭证内容**：
  - 收款人姓名、身份证号
  - 收入金额、税额、实发金额
  - 扣缴义务人（平台方）信息
  - 税务局备案编号
- **下载方式**：用户可在"提现记录"中下载电子凭证

## 四、自动化仲裁与惩戒算法

### 4.1 投诉-惩戒模型

#### 4.1.1 仲裁触发条件
- 申领人反馈"虚假线索"
- 申领人反馈"无法联系"且拨打次数≥10次
- 双方互相投诉
- 系统检测到异常行为

#### 4.1.2 仲裁判定逻辑

**步骤1：自动校验**
```
调用运营商API → 校验号码活性
  ↓
号码状态 = 正常 → 进入人工审核
号码状态 = 空号/停机 → 判定为虚假
```

**步骤2：违规定义**
- 判定为"干扰平台经营秩序"，而非简单的交易纠纷

**步骤3：仲裁结果**
- **判定真实**：积分解冻给分享人
- **判定虚假**：启动惩戒措施

#### 4.1.3 惩戒措施

**（1）积分扣除**
- **扣除顺序**：优先从分享人的"激励积分"账户中扣除
- **扣除倍数**：2-5倍违约金
- **计算公式**：
  ```
  违约金 = 应得激励积分 × 惩罚系数（2-5倍）
  ```

**（2）信用降权**
- **信誉分扣除**：根据违规严重程度扣分
- **功能限制**：降低信誉分，限制分享权限
- **提现审核**：延长提现审核期或暂停提现功能

### 4.2 信誉分管理 (Karma Score)

#### 4.2.1 动态评估
- **评估维度**：投诉率、分享成功率、反馈时效
- **实时计算**：根据用户行为动态调整
- **评分范围**：0-100分

#### 4.2.2 信誉分等级
- **90-100分**：优秀（优先推荐，提现绿色通道）
- **80-89分**：良好（正常权限）
- **70-79分**：一般（提现需人工审核）
- **60-69分**：较差（限制分享数量）
- **< 60分**：危险（暂停提现功能）

#### 4.2.3 封禁规则
- **自动封禁**：信誉分低于 40 分
- **封禁效果**：
  - 限制所有核心功能
  - 暂停提现功能
  - 需人工审核解封

#### 4.2.4 信誉分应用场景
- **提现审核**：90分以下无法提现
- **线索推荐**：高信誉分享人线索优先展示
- **申领优先**：高信誉用户可优先申领热门线索
- **佣金加成**：优秀信誉可获得额外积分奖励

## 五、交互流程大图 (Full Journey)

### 5.1 完整业务流程

```
1. 分享：
   用户 A 录入客户画像 → 签署承诺函 → 预检机制校验
   → 获得 50 个"激励积分"（待生效状态）

2. 大厅：
   用户 B 充值购买能量包（获得"充值积分"）
   → 在大厅浏览线索 → 选中用户 A 分享的画像

3. 申领：
   用户 B 消耗积分申领
   → 系统自动将该客户同步至用户 B 的【客户中心】
   → 来源标记为"线索引擎"

4. 跟进：
   用户 B 通过虚拟号联系客户
   → 在 CRM 更新进度
   → 客户进入【盘活客户列表】

5. 结算：
   用户 B 反馈真实 → 用户 A 的 50 积分转为"可用激励积分"
   用户 A 发起提现 → 系统校验积分为分享所得
   → 通过审核发放劳务费
```

### 5.2 用户甲（分享人）完整路径

```
1. 登录聚流App
   ↓
2. 点击"分享线索"按钮
   ↓
3. 填写客户画像（年龄段、职业、意向险种、保费预算、地域等）
   ↓
4. 勾选《信息来源合法及已获授权承诺函》
   ↓
5. 提交分享
   ↓
6. 系统预检校验（空号/实名校验）
   ↓
7. 校验通过 → 提示："分享成功，已获得50激励积分（待生效）"
   ↓
8. 线索进入大厅，被用户乙看到
   ↓
9. 用户乙查看详情后，支付积分申领
   ↓
10. 用户乙拨打电话跟进，3天后反馈"真实有效"
    ↓
11. 系统自动解冻50激励积分至用户甲账户
    ↓
12. 用户甲发起提现申请
    ↓
13. 系统校验：仅显示激励积分、身份一致、信誉分≥90分、无投诉
    ↓
14. 提现成功，平台代缴个税（金额<800元，无需缴税）
    ↓
15. 3个工作日后，款项到账银行卡
    ↓
16. 系统生成《劳务报酬结算凭证》，用户可下载
```

## 六、非功能性需求

### 6.1 性能需求
- **页面加载时间**：≤2秒
- **列表刷新时间**：≤1秒
- **支付响应时间**：≤3秒
- **并发用户数**：支持10000人同时在线
- **API响应时间**：95%的请求≤500ms

### 6.2 安全需求
- **数据加密**：客户手机号、身份证号等敏感信息加密存储
- **隐私保护**：虚拟号系统确保双方真实号码不泄露
- **权限控制**：基于角色的访问控制（RBAC）
- **防刷机制**：IP限频、设备指纹、行为分析
- **合规审计**：所有操作留痕，满足监管要求

### 6.3 可用性需求
- **界面友好**：符合保险代理人使用习惯，操作简单直观
- **引导完善**：新手引导、操作提示、错误提示清晰
- **离线支持**：部分功能支持离线操作，网络恢复后同步
- **多端一致**：iOS、Android、Web端体验一致
- **视觉规范**：黑色底色，品牌绿（#00bc71）作为核心行动点颜色

### 6.4 可维护性需求
- **模块化设计**：各功能模块解耦，便于迭代
- **日志完善**：详细的操作日志、错误日志
- **监控告警**：关键指标实时监控，异常自动告警
- **配置化管理**：积分规则、审核规则等支持后台配置

## 七、附录

### 7.1 术语表
- **分享人**：发布客户线索的保险代理人
- **浏览者**：浏览线索大厅的保险代理人
- **申领人**：成功申领并跟进线索的保险代理人
- **激励积分**：通过分享等行为获得，支持提现的积分（劳务报酬）
- **充值积分**：通过购买获得，仅用于消费的积分（技术服务费）
- **虚拟号**：保护隐私的中间号码，双向匿名
- **信誉分**：衡量用户信用等级的分数（0-100分）
- **客户中心**：私域客户管理系统（CRM）
- **盘活客户列表**：已建立联系并正在跟进的客户列表

### 7.2 数据字段清单

**客户画像字段**
- 年龄段（必填）
- 性别（必填）
- 职业（必填）
- 意向险种（必填）
- 保费预算（必填）
- 所在地域（必填）
- 客户意向程度（必填）
- 购买时间预期（选填）
- 客户购买习惯（选填）
- 备注信息（选填）

### 7.3 积分规则汇总表

| 操作 | 积分变动 | 积分类型 | 备注 |
|------|----------|----------|------|
| 分享线索 | +50（待生效） | 激励积分 | 反馈真实后转为可用 |
| 申领线索 | -N | 充值积分或激励积分 | 从总余额扣除 |
| 反馈真实 | +奖励 | 激励积分 | 鼓励反馈 |
| 虚假线索 | -2至-5倍 | 激励积分 | 仲裁后扣除 |
| 提现 | -对应积分 | 激励积分 | 仅支持激励积分提现 |

### 7.4 后续优化方向

2. **语音通话**：支持高质量语音通话与录音
3. **数据分析**：提供线索转化率、收益分析报表
4. **多语言支持**：支持港澳台及海外代理人
5. **客户中心深度集成**：更多CRM功能联动
