## 一、文档概述

### 1.1 产品定位
聚流 App 是一款面向保险代理人的客户经营互助平台，通过积分激励机制促进客户线索资源的合理流转与共享。

### 1.2 核心设计原则
- **行为去商业化**：以积分体系替代直接金钱交易
- **资金劳务化**：提现定义为"推广劳务报酬"
- **系统自治化**：自动化仲裁与信用约束机制

### 1.3 业务流程概览
客户线索流转遵循 **"分享 → 打听 → 申领"** 三段式架构，确保信息真实性与经营效率。

## 二、核心业务流程

### 2.1 客户线索分享 (Share)

#### 2.1.1 角色定义
- **分享人**：拥有客户线索资源但无法及时跟进或跨地域的保险代理人
- **目标用户**：需要将手头客户资源流转变现的代理人

#### 2.1.2 功能流程

```
开始 → 录入客户画像 → 签署承诺函 → 设置分享参数 → 提交成功
```

#### 2.1.3 详细功能规格

**（1）客户画像信息采集**
- **必填字段**：
  - 客户年龄段（如：25-30岁、30-35岁等）
  - 所属行业（下拉选择：金融、医疗、教育、互联网等）
  - 已购险种（多选：寿险、健康险、意外险、车险等）
  - 当前保险需求（下拉选择：教育金、养老、重疾保障等）
  - 家庭年收入区间（如：30-50万、50-80万等）
  - 客户意向程度（高/中/低）

- **选填字段**：
  - 购买时间预期（1个月内/3个月内/半年内/1年以上）
  - 客户购买习惯（线上偏好/线下偏好/无明显偏好）
  - 备注信息（特殊需求说明）

**（2）合规背书机制**
- **强制签署**：《信息来源合法及已获授权承诺函》
- **签署内容**：
  - 确认客户信息来源合法
  - 确认已获得客户授权用于保险咨询
  - 承诺不侵犯客户隐私权
  - 违约承担相应法律责任
- **签署方式**：电子签名 + 确认框勾选
- **法律效力**：系统记录签署时间与IP，具备法律追溯能力

**（3）分享参数设置**
- **最大允许打听人数**：系统默认10人，可自定义设置（范围1-50）
- **允许申领人数**：系统默认1人，可设置（范围1-3）
- **有效期限**：7天（系统自动下架）
- **是否匿名分享**：可选（勾选后大厅显示"匿名代理人"）

**（4）激励积分锁定**
- **积分预分配算法**：
  ```
  基础分值 × 画像完整度系数 × 需求热度系数 = 预分配激励积分
  ```
  - 基础分值：50积分
  - 画像完整度系数：0.8-1.2（根据填写字段完整度）
  - 需求热度系数：0.8-1.5（根据险种市场热度）

- **积分状态**：提交成功后，积分进入"冻结账户"，状态标记为"待确认"
- **积分类型标记**：系统自动标记为"激励积分-分享所得"，支持提现

**（5）界面交互流程**
1. 点击"分享线索"按钮进入分享页
2. 填写客户画像表单（带校验提示）
3. 勾选承诺函（未勾选无法提交）
4. 设置分享参数（智能推荐值）
5. 预览提交内容
6. 点击"提交分享"
7. 成功提示："分享成功，预计获得50激励积分（冻结中）"

### 2.2 客户线索大厅 (Browse)

#### 2.2.1 角色定义
- **浏览者**：对大厅画像感兴趣的意向被分享人
- **目标用户**：需要获取新客户资源的保险代理人

#### 2.2.2 大厅界面设计

**（1）筛选维度**
- **险种类型**：寿险、健康险、意外险、教育金、养老险等
- **客户年龄段**：25-30岁、30-35岁、35-40岁、40-45岁、45岁以上
- **家庭收入**：30万以下、30-50万、50-80万、80-100万、100万以上
- **需求紧急度**：高、中、低
- **更新时间**：今日、3日内、7日内
- **地理位置**：全国、省份、城市（基于分享人设置的地域标签）

**（2）列表展示字段**
- 客户画像卡片包含：
  - 年龄段标签
  - 行业标签
  - 已购险种标签
  - 需求标签（突出显示）
  - 收入区间
  - 分享人（或"匿名"）
  - 已打听人数/最大允许数
  - 已申领人数/允许申领数
  - 发布时间（如：2小时前、1天前）
  - 热度标识（高需求线索带🔥图标）

**（3）搜索功能**
- 关键词搜索：支持搜索行业、险种、需求关键词
- 高级搜索：组合筛选条件保存为常用搜索
- 排序方式：最新发布、热度最高、需求最急

#### 2.2.3 打听功能

**（1）打听入口**
- 点击线索卡片上的"打听详情"按钮
- 弹出打听确认框，显示打听费用（默认2积分）

**（2）在线对话机制**
- **支付方式**：消耗2积分（从个人积分账户扣除）
- **对话形式**：匿名在线对话窗口
- **消息规则**：
  - 双方匿名显示（仅显示"代理人A"和"代理人B"）
  - 消息自动保存7天
  - 支持文字消息（暂不支持图片、语音）
  - 消息发送后不可撤回

**（3）可询问内容（系统建议问题）**
- 客户是否方便接听陌生电话？
- 需求的真实性如何？有无购买记录或咨询历史？
- 客户对接电话的时间偏好（工作日/周末、上午/下午/晚上）？
- 是否有其他补充信息（如购买顾虑、特殊要求）？
- 客户已购产品的保单金额大概范围？

**（4）防骚扰机制**
- **有效时间**：单次打听对话窗口有效期24小时
- **外部联系方式禁止**：
  - 系统自动过滤手机号、微信号、QQ号等联系方式关键词
  - 检测到敏感信息自动提示："请勿交换外部联系方式"
  - 违规3次触发警告，5次暂停打听权限7天
- **打扰限制**：
  - 同一用户对同一线索最多发起1次打听
  - 分享人可手动关闭某个打听对话

**（5）打听成功流程**
```
浏览线索 → 点击打听 → 支付2积分 → 进入对话窗口 → 发起询问 → 分享人回复
→ 获得补充信息 → 评估是否申领
```

### 2.3 客户线索申领 (Claim)

#### 2.3.1 角色定义
- **申领人**：通过打听确认意向，决定正式经营该客户的用户
- **目标用户**：已完成信息核实，准备跟进的代理人

#### 2.3.2 申领权限获取

**（1）积分消耗规则**
- **基础申领费用**：48积分
- **费用计算逻辑**：
  ```
  申领费用 = 打听费用返还（2积分） + 线索价值（46积分）
  ```
  - 实际净消耗：48积分（但系统记录2积分用于打听，46积分用于申领）

- **积分不足处理**：
  - 余额不足时提示："积分不足，请先充值或完成其他任务"
  - 提供充值入口和积分任务入口

**（2）申领操作流程**
1. 在打听对话页或线索详情页点击"申领线索"按钮
2. 系统校验积分余额（≥48积分）
3. 显示申领确认框（含费用说明和经营保护条款）
4. 点击确认，扣除积分
5. 申领成功，进入线索经营页

#### 2.3.3 信息解密机制

**（1）解密信息内容**
- **客户姓氏**：显示（如："王先生"、"李女士"）
- **隐私拨号权限**：系统生成虚拟号码（如：400-888-1234）
  - 虚拟号与客户真实号码绑定
  - 通话记录系统自动保存
  - 虚拟号有效期7天（可续期）

- **不显示信息**：
  - 客户全名
  - 真实电话号码
  - 详细地址
  - 身份证号等敏感信息

**（2）拨号方式**
- **App内拨号**：点击"拨打"按钮，系统自动呼叫虚拟号
- **通话记录**：系统记录通话时长、拨打次数
- **隐私保护**：客户来电显示也是虚拟号，双向匿名

#### 2.3.4 经营保护期

**（1）独占保护规则**
- **保护期限**：申领成功后7天
- **隐藏机制**：该线索在大厅对其他用户完全隐藏
- **续约机制**：7天后如未反馈结果，系统自动延长保护期3天并发送提醒

**（2）保护期内操作**
- 申领人可无限次拨打虚拟号
- 可在系统内记录跟进笔记（仅自己可见）
- 可标记经营状态（跟进中/已成交/已放弃）

#### 2.3.5 反馈闭环机制

**（1）强制反馈要求**
- **反馈时限**：申领成功后48小时内必须反馈
- **反馈选项**：
  - 真实有效：客户信息属实，已建立联系
  - 虚假信息：客户不存在或需求不真实
  - 无法联系：拨打10次以上未接通
  - 其他原因：填写具体说明

**（2）反馈触发的系统动作**

**情况A：反馈"真实有效"**
- 分享人账户的激励积分由"冻结"转为"可用"
- 申领人获得10积分奖励（鼓励反馈）
- 线索状态变更为"已完成"，双方评价系统开放

**情况B：反馈"虚假信息"**
- 启动自动仲裁流程（见4.2节）
- 暂停分享人该线索的积分发放
- 记录投诉，进入人工审核队列

**情况C：反馈"无法联系"**
- 分享人积分继续冻结，等待二次确认
- 系统自动标记该线索为"联系困难"
- 给予24小时二次反馈期

**情况D：超时未反馈**
- 系统自动发送提醒（24小时、36小时、48小时）
- 48小时后自动标记为"反馈超时"
- 分享人积分继续冻结，等待人工仲裁

**（3）评价系统**
- 双方可互评（1-5星）
- 评价影响信誉分（见4.3节）
- 评价内容仅平台管理员可见（防止恶意差评）

## 三、积分体系设计

### 3.1 积分类型定义

#### 3.1.1 充值积分
- **来源**：用户购买"能量包"（如：10元=100积分）
- **标记属性**：不可提现、仅用于消费
- **使用场景**：打听线索、申领线索、购买增值服务
- **过期规则**：无有效期限制

#### 3.1.2 激励积分
- **来源1**：线索分享且被评价为真实后获得
- **来源2**：邀请新用户注册并完成首单
- **来源3**：参与平台活动获得奖励
- **标记属性**：支持提现（需满足条件）
- **使用场景**：所有消费场景 + 提现
- **过期规则**：12个月未使用自动过期

#### 3.1.3 活动赠送积分
- **来源**：平台促销活动、节日礼包
- **标记属性**：不可提现、有使用限制
- **使用场景**：仅限指定活动使用
- **过期规则**：根据活动规则设定（通常7-30天）

### 3.2 积分账户管理

#### 3.2.1 账户结构
```
总积分余额 = 充值积分 + 激励积分 + 活动赠送积分
可用积分余额 = 总余额 - 冻结积分
提现可用余额 = 激励积分余额 - 冻结激励积分
```

#### 3.2.2 冻结机制
- **分享冻结**：分享线索后，预分配的激励积分进入冻结账户
- **仲裁冻结**：争议线索的积分暂时冻结，等待仲裁结果
- **提现冻结**：提现申请提交后，对应积分冻结直至到账

#### 3.2.3 积分明细记录
- 每笔积分变动生成流水记录
- 记录字段：时间、类型、来源/去向、数量、余额、备注
- 用户可随时查看积分明细

### 3.3 提现校验逻辑

#### 3.3.1 双账户校验机制
```
提现金额 ≤ 可用激励积分余额
提现金额 > 0
提现金额为整数（不支持小数）
单次提现限额：50-5000积分
每日提现限额：10000积分
每月提现限额：50000积分
```

#### 3.3.2 身份一致性校验
- **实名认证**：用户必须完成实名认证（身份证+人脸识别）
- **银行卡绑定**：提现银行卡必须与实名信息一致
- **三位一体校验**：
  ```
  提现银行卡姓名 == 实名认证姓名 == App注册主体
  三者必须完全一致，否则提现失败
  ```

#### 3.3.3 信用硬约束
- **信誉分要求**：信誉分 ≥ 90分（满分100分）
- **投诉检查**：无未处理的投诉或仲裁案件
- **冻结检查**：无被冻结的提现申请
- **违规记录**：近30天无严重违规记录

#### 3.3.4 提现流程
```
1. 用户点击"提现" → 2. 系统校验可用激励积分
3. 填写提现金额 → 4. 系统校验身份与信用
5. 确认提现信息 → 6. 提交提现申请
7. 系统冻结对应积分 → 8. 生成提现订单
9. 平台审核（自动+人工） → 10. 财务打款
11. 用户确认到账 → 12. 订单完成
```

**提现时间**：提交后3-5个工作日到账

### 3.4 税务合规处理

#### 3.4.1 账单定义
- **充值账单**：定义为"技术服务费"
- **提现账单**：定义为"推广劳务报酬"

#### 3.4.2 个税代扣代缴
- **起征点**：单次提现金额 > 800元（对应积分 > 8000积分）
- **税率**：按劳务报酬税率（20%、30%、40%累进）
- **自动计算**：系统自动计算应扣税额
- **显示明细**：
  ```
  申请提现：1000元
  个税扣除：200元
  实际到账：800元
  ```

#### 3.4.3 凭证生成
- **自动生成**：《劳务报酬结算凭证》
- **凭证内容**：
  - 收款人姓名、身份证号
  - 收入金额、税额、实发金额
  - 扣缴义务人（平台方）信息
  - 税务局备案编号
- **下载方式**：用户可在"提现记录"中下载电子凭证

## 四、平台治理机制

### 4.1 自动化仲裁算法

#### 4.1.1 仲裁触发条件
- 申领人反馈"虚假信息"
- 申领人反馈"无法联系"且拨打次数≥10次
- 双方互相投诉
- 系统检测到异常行为（如：同一号码多次分享）

#### 4.1.2 仲裁判定逻辑

**步骤1：自动校验**
```
调用运营商API → 校验号码状态
  ↓
号码状态 = 正常 → 进入人工审核
号码状态 = 空号/停机 → 判定为虚假
号码状态 = 频繁呼叫 → 调取通话录音
```

**步骤2：人工审核**
- 平台客服介入调查
- 查看双方聊天记录、通话记录
- 联系分享人核实情况
- 必要时电话回访客户

**步骤3：仲裁结果**
- **判定真实**：积分解冻给分享人，申领人承担仲裁费用（10积分）
- **判定虚假**：积分扣除给申领人，分享人承担违约金
- **无法判定**：积分退回，双方信誉分各扣5分

#### 4.1.3 惩戒措施

**虚假信息发布惩罚**
- **第一次违规**：扣除2倍激励积分（从冻结账户扣除）
- **第二次违规**：扣除5倍激励积分 + 信誉分-20
- **第三次违规**：永久封禁账号 + 列入黑名单

**违约金扣除规则**
```
违约金 = 应得激励积分 × 惩罚系数（2-5倍）
优先扣除顺序：激励积分 > 活动积分 > 充值积分
```

**冻结账户处理**
- 仲裁期间，分享人该线索的积分冻结
- 仲裁结束后根据结果执行解冻或扣除

### 4.2 信誉分体系

#### 4.2.1 信誉分构成
```
基础分：100分
+ 线索真实加分（每次+5分，上限+20分）
- 虚假线索扣分（每次-10分，无下限）
- 投诉成立扣分（每次-5分）
- 超时反馈扣分（每次-2分）
- 仲裁败诉扣分（每次-10分）
```

#### 4.2.2 信誉分等级
- **90-100分**：优秀（优先推荐，提现绿色通道）
- **80-89分**：良好（正常权限）
- **70-79分**：一般（提现需人工审核）
- **60-69分**：较差（限制分享数量）
- **< 60分**：危险（暂停提现功能）

#### 4.2.3 信誉分应用场景
- **提现审核**：90分以下无法提现
- **线索推荐**：高信誉分享人线索优先展示
- **申领优先**：高信誉用户可优先申领热门线索
- **佣金加成**：优秀信誉可获得额外10%激励积分

### 4.3 反作弊机制

#### 4.3.1 虚假线索识别
- **号码库比对**：同一号码30天内不允许重复分享
- **画像一致性检查**：同一分享人多次分享相似画像触发预警
- **申领成功率监控**：分享人线索申领成功<30%触发审核
- **反馈率监控**：长期无反馈记录的分享人降权

#### 4.3.2 刷积分行为检测
- **账号关联检测**：同一设备多账号、同一银行卡多账号
- **异常交易监控**：短时间内大量积分充值/提现
- **互相刷单检测**：A分享、B申领、C分享、A申领的循环模式
- **IP异常检测**：同一IP大量注册或操作

#### 4.3.3 处罚措施
- **警告**：首次发现发送警告通知
- **降权**：限制功能使用，降低推荐权重
- **冻结**：暂时冻结账号7-30天
- **封禁**：永久封禁账号，清零积分

## 六、非功能性需求

### 6.1 性能需求
- **页面加载时间**：≤2秒
- **列表刷新时间**：≤1秒
- **支付响应时间**：≤3秒
- **并发用户数**：支持10000人同时在线
- **API响应时间**：95%的请求≤500ms

- **数据加密**：客户手机号、身份证号等敏感信息加密存储
- **隐私保护**：虚拟号系统确保双方真实号码不泄露
- **权限控制**：基于角色的访问控制（RBAC）
- **防刷机制**：IP限频、设备指纹、行为分析
- **合规审计**：所有操作留痕，满足监管要求

- **界面友好**：符合保险代理人使用习惯，操作简单直观
- **引导完善**：新手引导、操作提示、错误提示清晰
- **离线支持**：部分功能支持离线操作，网络恢复后同步
- **多端一致**：iOS、Android、Web端体验一致

### 6.4 可维护性需求

- **日志完善**：详细的操作日志、错误日志
