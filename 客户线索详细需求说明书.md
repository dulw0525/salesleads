## 一、产品定位与核心理念

### 1.1 产品定位
聚流 App 是一款赋能保险代理人的展业互助与客户经营平台，构建闭环、合规且高效的客户线索流转生态。

### 1.2 核心逻辑
建立基于"聚流积分"的资源分享机制，将传统的"线索交易"重塑为"合规分享与劳务激励"。

### 1.3 合规基石
- **业务与财务解耦**：操作与资金流动分离管理
- **分类积分管理**：不可提现积分（充值）与可提现积分（激励）严格区分
- **隐私拨号隔离**：虚拟号系统保护双方隐私
- **劳务报酬定性**：提现定义为劳务报酬，合规税务处理

## 二、客户线索全生命周期三大模块

### 2.1 客户线索分享 (Share)

#### 2.1.1 录入画像
- **必填字段**：
  - 客户年龄段（25-30岁、30-35岁、35-40岁、40-45岁、45岁以上）
  - 性别
  - 职业
  - 意向险种（寿险、健康险、意外险、教育金、养老险等）
  - 保费预算区间（1-5万、5-10万、10-20万、20万以上）
  - 所在地域（省份/城市）
  - 客户意向程度（高/中/低）

- **选填字段**：
  - 购买时间预期（1个月内/3个月内/半年内/1年以上）
  - 客户购买习惯（线上偏好/线下偏好/无明显偏好）
  - 备注信息

#### 2.1.2 合规背书
- **强制签署**：《信息来源合法及已获授权承诺函》
- **签署内容**：
  - 确认客户信息来源合法
  - 确认已获得客户授权用于保险咨询
  - 承诺不侵犯客户隐私权
  - 违约承担相应法律责任
- **签署方式**：电子签名 + 确认框勾选
- **法律效力**：系统记录签署时间与IP，具备法律追溯能力

#### 2.1.3 自动化预检
- **空号/实名校验**：后端调用 API 进行号码活性静态校验
- **校验结果**：
  - 校验通过：进入正常分享流程
  - 校验失败：提示用户重新录入或放弃分享

#### 2.1.4 管理台（我的分享）

**（1）增删改查权限**
- **可编辑/删除状态**：
  - 草稿
  - 待发布
  - 待整改

- **禁止修改/删除状态**：
  - 已申领
  - 已结算

**（2）状态追踪**
- 实时监控线索状态：
  - **公示中**：线索在大厅展示中
  - **已申领**：线索已被他人申领
  - **待整改**：线索因申诉被退回待整改
  - **已结算**：线索已完成确认，积分已发放

**（3）界面功能**
- 支持按状态筛选线索
- 显示线索当前状态标签
- 提供编辑、删除、查看详情操作入口

### 2.2 客户线索大厅 (Marketplace)

#### 2.2.1 脱敏展示
- **展示内容**：
  - 意向画像（年龄段、职业、险种、预算等标签）
  - 倒计时有效时间
  - 分享人信誉分
  - 分享人好评率
- **不展示内容**：
  - 客户真实姓名
  - 客户电话号码
  - 客户详细地址
  - 其他敏感信息

#### 2.2.2 多维筛选
- **险种标签**：寿险、健康险、意外险、教育金、养老险等
- **地域筛选**：全国、省份、城市
- **预算区间**：1-5万、5-10万、10-20万、20万以上
- **更新时间**：今日、3日内、7日内
- **支持组合筛选**：多条件交叉检索

#### 2.2.3 视觉规范
- **整体风格**：黑色科技风，延续 AI 科技风格
- **组件库**：shadcn/ui
- **核心引导色**：品牌绿（#00bc71）
- **卡片设计**：沉浸式卡片布局，信息清晰易读

#### 2.2.4 排序方式
- 最新发布（默认）
- 热度最高
- 价格最低
- 倒计时最短

### 2.3 客户线索申领与确认 (Claim & Confirm)

#### 2.3.1 申领锁分
- **积分锁定**：点击申领后，申领人账户对应的"充值积分"进入**锁定状态**
- **锁定原因**：防止申领过程中积分变动
- **解锁条件**：确认成功或申诉成功后解锁

#### 2.3.2 CRM 自动化同步

**（1）自动同步机制**
- 申领成功后，系统自动将客户画像同步至申领人的**【客户中心】**
- 同步字段：年龄段、职业、意向险种、保费预算、地域、意向程度

**（2）来源标记**
- 客户来源字段自动锁定为**"线索引擎"**
- 不可修改，确保数据溯源

**（3）盘活追踪**
- 一旦申领人发起有效跟进，客户将自动进入**【盘活客户列表】**
- 有效跟进定义：拨打电话、添加备注、更新状态等

#### 2.3.3 隐私通讯
- **虚拟号/隐私拨号**：通过内置虚拟号联系客户
- **双向匿名**：真实手机号不显示、不落地
- **虚拟号生成**：系统自动生成中间号码
- **通话记录**：系统自动保存通话时长、拨打次数

#### 2.3.4 信息确认逻辑

**（1）手动确认**
- **操作方式**：申领人核实正确后点击"确认信息正确"
- **触发结果**：立即触发积分结算，分享人积分解冻

**（2）催办提醒**
- **提醒时机**：T+3 天未确认
- **提醒方式**：系统发送 Push 消息
- **提醒内容**："您申领的客户即将过期，请及时在客户中心确认"

**（3）自动确认**
- **执行时机**：T+7 天未确认且未申诉
- **执行方式**：系统执行**强制自动确认**
- **通知双方**：向分享人和申领人发送确认通知
- **积分结算**：自动完成积分结算

## 三、积分体系与财务合规

### 3.1 积分类型定义

系统对积分进行底层属性标记，确保资金流向的法律安全性。

| 积分属性 | 获取来源 | 提现权限 | 核心定义 |
| :---- | :---- | :---- | :---- |
| **充值积分** | 人民币充值购买"能量包" | **禁止提现** | 平台技术服务费，仅限站内消耗 |
| **激励积分** | 分享线索被确认后解冻 | **支持提现** | 平台支付给用户的**劳务推广报酬** |

#### 3.1.1 充值积分
- **来源**：用户购买"能量包"（如：10元=100积分）
- **标记属性**：不可提现、仅用于消费
- **使用场景**：申领线索、购买增值服务
- **过期规则**：无有效期限制

#### 3.1.2 激励积分
- **来源**：分享线索被确认后解冻（预分配50积分）
- **标记属性**：支持提现（需满足条件）
- **使用场景**：所有消费场景 + 提现
- **过期规则**：12个月未使用自动过期

### 3.2 积分账户管理

#### 3.2.1 账户结构
```
总积分余额 = 充值积分 + 激励积分
可用积分余额 = 总余额 - 冻结积分
提现可用余额 = 激励积分余额 - 冻结激励积分
```

#### 3.2.2 冻结机制
- **分享冻结**：分享线索后，预分配的激励积分进入冻结账户
- **申领锁定**：申领过程中，积分进入锁定状态
- **仲裁冻结**：争议线索的积分暂时冻结，等待仲裁结果
- **提现冻结**：提现申请提交后，对应积分冻结直至到账

### 3.3 提现校验逻辑

#### 3.3.1 来源属性校验
- **提取限制**：提现时系统仅统计并允许提取"激励积分"余额
- **充值积分**：无法被选中用于提现
- **自动过滤**：系统自动区分积分类型

#### 3.3.2 合规流程
- **劳务协议**：提现需签署《劳务合作协议》
- **个税代扣**：平台代扣代缴个税
- **结算凭证**：生成《劳务报酬结算凭证》

#### 3.3.3 身份一致性校验
- **实名认证**：用户必须完成实名认证（身份证+人脸识别）
- **银行卡绑定**：提现银行卡必须与实名信息一致
- **三位一体校验**：
  ```
  提现银行卡姓名 == 实名认证姓名 == App注册主体
  三者必须完全一致，否则提现失败
  ```

#### 3.3.4 提现流程
```
1. 用户点击"提现" → 2. 系统仅显示可用激励积分
3. 填写提现金额 → 4. 系统校验身份与信用
5. 确认提现信息 → 6. 提交提现申请
7. 系统冻结对应积分 → 8. 生成提现订单
9. 平台审核（自动+人工） → 10. 财务打款
11. 用户确认到账 → 12. 订单完成
```

**提现时间**：提交后3-5个工作日到账

### 3.4 税务合规处理

#### 3.4.1 账单定义
- **充值账单**：定义为"技术服务费"
- **提现账单**：定义为"推广劳务报酬"

#### 3.4.2 个税代扣代缴
- **起征点**：单次提现金额 > 800元
- **税率**：按劳务报酬税率（20%、30%、40%累进）
- **自动计算**：系统自动计算应扣税额
- **显示明细**：
  ```
  申请提现：1000元
  个税扣除：200元
  实际到账：800元
  ```

#### 3.4.3 凭证生成
- **自动生成**：《劳务报酬结算凭证》
- **凭证内容**：
  - 收款人姓名、身份证号
  - 收入金额、税额、实发金额
  - 扣缴义务人（平台方）信息
  - 税务局备案编号
- **下载方式**：用户可在"提现记录"中下载电子凭证

## 四、异常处理与自动化仲裁

### 4.1 申诉与熔断机制

#### 4.1.1 发起申诉
- **申诉条件**：申领人发现信息有误（空号、非本人等）
- **申诉方式**：提交申诉理由并上传举证材料
- **申诉内容**：
  - 申诉类型（空号/非本人/信息不符等）
  - 具体说明
  - 举证材料（通话记录、聊天截图等）

#### 4.1.2 申诉处理
- **处理流程**：
  1. 申领人提交申诉
  2. 系统接收并标记为"申诉中"
  3. 自动/人工审核申诉材料
  4. 审核结果通知双方

- **申诉成功**：
  - 申领人积分原路退回
  - 线索从其 CRM 移除
  - 线索退回分享人"待整改"列表

- **申诉失败**：
  - 积分不予退回
  - 线索状态保持"已申领"
  - 提醒申领人确认信息

#### 4.1.3 退回整改
- **状态标记**：线索标记为"异常"且**禁止再次直接上架**
- **整改列表**：线索退回到分享人的**【待整改】**列表
- **整改要求**：分享人必须修改信息并重新通过静态 API 校验

#### 4.1.4 重启发布
- **修改信息**：分享人编辑线索信息
- **重新校验**：调用 API 重新进行空号/实名校验
- **校验通过**：可再次发起分享
- **校验失败**：提示用户修改，不允许发布

### 4.2 自动化惩戒

#### 4.2.1 违规判定
- **判定标准**：判定为"干扰平台秩序"
- **判定依据**：
  - 申诉成功率
  - 客户号码活性
  - 历史违规记录

#### 4.2.2 积分扣除
- **扣除顺序**：优先从分享人的"激励积分"余额中扣除
- **扣除倍数**：2-5 倍积分（根据违规严重程度）
- **计算公式**：
  ```
  违约金 = 应得激励积分 × 惩罚系数（2-5倍）
  ```

#### 4.2.3 信誉分（Karma Score）
- **评分范围**：0-100分
- **扣分规则**：根据违规严重程度扣除相应分数
- **封禁规则**：
  - 信誉分低于阈值将触发限流
  - 严重违规触发永久封号

## 五、消息通知矩阵

### 5.1 通知事件表

| 事件节点 | 通知对象 | 消息核心内容 |
| :---- | :---- | :---- |
| **线索被领用** | 分享用户 | "您的线索 [客户姓氏] 已被领用，积分已冻结。" |
| **系统催办 (T+3)** | 申领用户 | "您申领的客户即将过期，请及时在客户中心确认。" |
| **自动确认 (T+7)** | 双方用户 | "系统已自动确认信息准确，积分结算完成。" |
| **申诉退回** | 分享用户 | "您的分享因申诉成功被退回待整改列表，请及时处理。" |

### 5.2 通知渠道
- **Push 推送**：实时推送重要事件
- **站内消息**：消息中心保存所有通知记录
- **短信提醒**：关键事件发送短信（可选）

### 5.3 通知优先级
- **高优先级**：线索被领用、申诉退回
- **中优先级**：系统催办、积分到账
- **低优先级**：日常提醒、系统公告

## 六、交互路径全景图

### 6.1 完整业务流程

```
1. 分享：
   分享人在"我的分享"录入客户画像
   → 签署承诺函
   → 系统预检校验
   → 获得冻结积分（50激励积分）

2. 申领：
   申领人在大厅浏览线索
   → 点击申领
   → 充值积分进入锁定状态
   → 申领成功
   → 数据自动入库 CRM（来源：线索引擎）

3. 核实：
   申领人通过虚拟号联系客户
   ↓
   ┌─────────────────────────────────────┐
   │         路径分支                    │
   └─────────────────────────────────────┘
   ↓                                     ↓
   路径 A：                             路径 B：
   申领人点击"确认信息正确"           申领人发起申诉
   （或 7 天自动确认）                 ↓
   ↓                              申诉成功
   分享人积分解冻并转正              ↓
   ↓                              申领人积分退回
   分享人可申请提现                  ↓
                                  线索退回"待整改"列表
                                  ↓
                                  分享人修改信息
                                  重新校验后发布

4. 转化：
   分享人申请提现激励积分
   → 签署《劳务合作协议》
   → 平台代扣个税
   → 劳务费到账银行卡
```

15. 进入"客户中心"
    ↓
16. 点击"拨打"按钮
    ↓
17. 系统呼叫虚拟号
    ↓
18. 与客户通话，确认需求真实性
    ↓
19. 选择操作路径：
    ↓
    ┌─────────────────────────────────┐
    │     分支选择                    │
    └─────────────────────────────────┘
    ↓                                 ↓
    路径A：确认真实                  路径B：发现异常
    ↓                                 ↓
    点击"确认信息正确"              点击"发起申诉"
    ↓                                 ↓
    积分结算完成                     填写申诉理由
    ↓                                 上传举证材料
    获得10积分奖励                   ↓
    ↓                             申诉成功
    客户进入【盘活客户列表】         ↓
    ↓                             积分原路退回
    继续跟进客户                     线索从CRM移除
    完成保单销售

## 七、非功能性需求

### 7.1 性能需求
- **页面加载时间**：≤2秒
- **列表刷新时间**：≤1秒
- **支付响应时间**：≤3秒
- **并发用户数**：支持10000人同时在线
- **API响应时间**：95%的请求≤500ms

### 7.2 安全需求
- **数据加密**：客户手机号、身份证号等敏感信息加密存储
- **隐私保护**：虚拟号系统确保双方真实号码不泄露
- **权限控制**：基于角色的访问控制（RBAC）
- **防刷机制**：IP限频、设备指纹、行为分析
- **合规审计**：所有操作留痕，满足监管要求

### 7.3 可用性需求
- **界面友好**：符合保险代理人使用习惯，操作简单直观
- **引导完善**：新手引导、操作提示、错误提示清晰
- **离线支持**：部分功能支持离线操作，网络恢复后同步
- **多端一致**：iOS、Android、Web端体验一致
- **视觉规范**：黑色科技风，品牌绿（#00bc71）作为核心引导色

### 7.4 可维护性需求
- **模块化设计**：各功能模块解耦，便于迭代
- **日志完善**：详细的操作日志、错误日志
- **监控告警**：关键指标实时监控，异常自动告警
- **配置化管理**：积分规则、审核规则等支持后台配置

## 八、附录

### 8.1 术语表
- **分享人**：发布客户线索的保险代理人
- **申领人**：成功申领并跟进线索的保险代理人
- **充值积分**：通过购买获得，仅用于消费的积分（技术服务费）
- **激励积分**：通过分享获得，支持提现的积分（劳务报酬）
- **虚拟号**：保护隐私的中间号码，双向匿名
- **信誉分**：衡量用户信用等级的分数（0-100分）
- **客户中心**：私域客户管理系统（CRM）
- **盘活客户列表**：已建立联系并正在跟进的客户列表
- **线索引擎**：客户线索功能模块，作为客户来源标识

### 8.2 数据字段清单

**客户画像字段**
- 年龄段（必填）
- 性别（必填）
- 职业（必填）
- 意向险种（必填）
- 保费预算（必填）
- 所在地域（必填）
- 客户意向程度（必填）
- 购买时间预期（选填）
- 客户购买习惯（选填）
- 备注信息（选填）

### 8.3 积分规则汇总表

| 操作 | 积分变动 | 积分类型 | 备注 |
|------|----------|----------|------|
| 分享线索 | +50（冻结） | 激励积分 | 确认后转为可用 |
| 申领线索 | -N（锁定） | 充值积分 | 确认后扣除 |
| 确认真实 | +10 | 激励积分 | 鼓励反馈 |
| 申诉成功 | +N | 充值积分 | 积分退回 |
| 虚假线索 | -2至-5倍 | 激励积分 | 仲裁后扣除 |
| 提现 | -对应积分 | 激励积分 | 仅支持激励积分提现 |

3. **数据分析**：提供线索转化率、收益分析报表
4. **多语言支持**：支持港澳台及海外代理人
5. **客户中心深度集成**：更多CRM功能联动
