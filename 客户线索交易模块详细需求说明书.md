# 保险代理人展业平台
## 客户线索交易模块详细需求说明书

**文档版本**: V1.0
**创建日期**: 2026年3月11日
**产品定位**: 保险代理人展业AI赋能平台 - 客户线索交易平台

---

## 一、项目概述

### 1.1 业务背景
为保险代理人提供一个去中心化的、透明的保险意向线索流转市场,通过交易高价值销售线索促进用户活跃,带动平台其他经营工具的使用。

### 1.2 核心价值
- **对卖家**: 快速变现闲置客户资源,0手续费,灵活提现
- **对买家**: 获取精准意向客户线索,提高成交转化率
- **对平台**: 构建业务闭环,增强用户粘性

### 1.3 业务流程概览
```
卖家上传(含AI校验)
    ↓
市场展示(脱敏处理)
    ↓
买家购买(积分结算)
    ↓
资金冻结(24小时售后保护期)
    ↓
售后期结束 → 提现(税务代缴)
```

### 1.4 关键指标
- 平台不收取撮合手续费
- 24小时售后保护期
- 买卖双方实名认证率100%
- AI防撞单查重准确率≥99%

## 三、功能需求详述

### 3.1 用户准入与实名风控模块

#### 3.1.1 基础认证流程
**功能描述**: 所有用户必须完成三要素实名认证方可进入交易大厅

**认证要素**:
- 姓名(与身份证一致)
- 身份证号码
- 手机号(接收验证码)

**流程设计**:
```
1. 用户进入交易模块
2. 系统检测认证状态
3. 未认证 → 跳转认证页面
4. 填写三要素信息
5. 调用第三方实名认证接口核验
6. 核验成功 → 更新用户状态为"已认证"
7. 核验失败 → 提示错误信息,允许重新认证
```

**数据存储**:
- 用户认证状态标记
- 身份证信息(加密存储)
- 手机号(加密存储)
- 认证时间戳

#### 3.1.2 高级认证流程
**功能描述**: 发布线索和提现前需通过人脸识别(活体检测)

**触发场景**:
- 首次发布线索
- 首次申请提现
- 系统检测到异常登录

**流程设计**:
```
1. 用户触发需要高级认证的操作
2. 系统判断是否已完成高级认证
3. 未认证 → 弹出人脸识别窗口
4. 用户进行活体检测(眨眼/摇头等动作)
5. 调用人脸识别接口比对身份信息
6. 比对成功 → 更新认证状态,继续操作
7. 比对失败 → 提示重新认证(最多3次)
```

#### 3.1.3 合规协议签署

##### 《线索来源合法合规承诺书》
**签署时机**: 首次发布线索前
**核心条款**:
- 承诺线索来源合法合规
- 承诺已获得客户授权或客户为公开可获取信息
- 违反承诺承担相应法律责任

##### 《个人众包服务协议》
**签署时机**: 首次提现前
**核心条款**:
- 个人作为独立服务提供方
- 平台通过灵活用工平台进行税务处理
- 了解并接受预扣费用机制

**签署方式**:
- 电子签名
- 协议版本管理
- 签署时间记录

### 3.2 线索上架与AI校验模块(卖家端)

#### 3.2.1 线索信息录入

**必填字段**:
| 字段 | 类型 | 说明 |
|------|------|------|
| 客户姓名 | 文本 | 真实姓名 |
| 联系方式 | 手机号 | 11位手机号,加密存储 |
| 城市 | 下拉选择 | 省市区三级联动 |
| 年龄 | 整数 | 岁数 |
| 职业 | 下拉选择 | 从预设职业列表选择 |
| 预计保费预算 | 整数 | 单位:元 |

**意向分级(单选)**:
- 暂无意向(了解阶段)
- 潜在了解(有初步兴趣)
- 明确规划(已有购买计划)
- 近期急需(1-3个月内购买)

**选填字段**:
- 险种意向(多选): 人寿保险、健康保险、财产保险、车险等
- 备注说明: 额外信息补充

**界面设计要求**:
- 表单分步填写,降低操作复杂度
- 实时表单验证
- 保存草稿功能

#### 3.2.2 自定义定价

**定价规则**:
- 卖家自主输入价格
- 单位: 展业分
- 价格范围: 10-5000分
- 最小步进: 1分

**定价参考**:
- 系统根据线索画像提供"推荐价格范围"
- 显示近期同类线索成交均价
- 显示价格竞争力提示

#### 3.2.3 AI自动化校验

##### 防撞单校验
**规则**:
- 48小时内同一手机号禁止重复上架
- 校验维度: 手机号(脱敏后比对)
- 校验范围: 所有已上架和成交线索

**处理逻辑**:
```
1. 卖家提交线索
2. 系统提取手机号并脱敏
3. 查询数据库48小时内相同脱敏号记录
4. 存在重复 → 拦截并提示"该号码已有人上架"
5. 不存在重复 → 通过校验
```

**提示文案**: "该客户联系方式在48小时内已被其他代理人上架,为保护客户体验,禁止重复发布。"

##### 空号检测
**触发时机**: 线索上架瞬间
**检测接口**: 调用第三方运营商空号检测API
**检测维度**:
- 停机号码
- 空号
- 无效号码

**处理逻辑**:
```
1. 提交线索时触发空号检测
2. 调用空号检测API
3. API返回结果:
   - 正常号码 → 继续发布流程
   - 异常号码 → 阻断发布,提示"该号码状态异常,请核实"
```

**错误提示**: "检测到该手机号为停机/空号,请核实客户联系方式后再发布。"

##### AI价值预估
**输入参数**:
- 年龄段
- 职业类型
- 预计保费预算
- 险种意向
- 意向分级

**算法逻辑**:
- 基于历史成交数据训练的机器学习模型
- 综合评估线索质量得分(0-100分)
- 根据得分映射到推荐价格区间

**输出结果**:
- 推荐价格下限
- 推荐价格上限
- 价值评估说明(如:"高价值线索,历史成交率85%")

**界面展示**:
```
您的线索价值评估: 88分 (优秀)
推荐定价区间: 150-300展业分
历史同类线索平均成交价: 220展业分
```

#### 3.2.4 线索上架审核

**自动审核规则**:
- 通过所有AI校验 → 自动上架
- 任一校验失败 → 拦截并提示原因

**人工审核场景**:
- 系统标记为高风险线索
- 涉及敏感行业或大额保费
- 卖家信用分低于阈值

**上架成功反馈**:
- 上架成功提示
- 线索编号
- 预计曝光位置
- 定价合理性评价

### 3.3 线索交易大厅(买家端)

#### 3.3.1 线索列表展示

**脱敏展示规则**:
- 手机号: 显示为 138****8888 格式
- 姓名: 完全隐藏
- 其他信息: 正常展示

**展示字段**:
- 区域: 省市信息(如:北京·朝阳区)
- 年龄段: 显示区间(如:30-40岁)
- 职业: 完整显示
- 预算范围: 显示区间(如:10-20万)
- 险种意向: 标签化展示
- 意向等级: 图标+文字
- 价格: XX展业分
- 上架时间: 相对时间(如:2小时前)
- 卖家信用: 星级评分
- AI评分: 0-100分

**筛选条件**:
- 区域(省市区)
- 年龄范围
- 预算范围
- 险种意向(多选)
- 意向等级
- 价格区间
- 上架时间

**排序方式**:
- 默认: 综合推荐(智能排序)
- 价格从低到高
- 价格从高到低
- 最新上架

**脱敏详情**:
- 客户画像: 年龄、职业、预算、险种意向

- AI评分及推荐理由

**操作按钮**:
- 收藏(爱心图标)

**推荐算法**:
- 基于买家历史购买偏好

**推荐场景**:
- 首页"为你推荐"
- 购买后"猜你喜欢"
- 浏览相似线索时的"相关推荐"

**推荐标签**:
- "智能推荐"
- "高成交率"
- "性价比优选"

#### 3.3.4 购买流程

**流程设计**:
```
1. 买家点击"购买"按钮
2. 系统检查买家账户余额

4. 扣除买家"充值分"账户
5. 线索状态变更为"已售出"

7. 解锁线索真实联系方式
8. 生成24小时售后保护期倒计时
9. 跳转至联系页面
```

```
价格: 200展业分
剩余余额: 1800展业分
[取消] [确认购买]
```

**购买成功提示**:

```
真实联系方式已解锁

[拨打电话] [复制号码]
```

### 3.5 售后保障与纠纷处理模块

#### 3.5.1 担保交易机制

**资金冻结规则**:
- 购买成功后,积分进入卖家"待结算"账户
- 冻结时长: 24小时
- 冻结期间: 卖家不可提现,但可查看

**状态流转**:
```
待结算(24小时内)
    ↓ 无申诉
可提现(24小时后)
    ↓ 申请提现
提现中(处理中)
    ↓ 处理完成
已提现
```

#### 3.5.2 申诉流程设计

##### 硬性错误申诉(空号/错号/非本人)

**申诉入口**:
- 已购买线索详情页的"申诉"按钮
- 订单列表页的"申请售后"入口

**申诉类型**:
- 空号(拨打提示停机/空号)
- 错号(非本人接听)
- 非本人(接听者非目标客户)

**自动化处理流程**:
```
1. 买家选择申诉类型
2. 提交申诉(无需上传证据)
3. 系统自动调用运营商接口核实
4. 核实结果:
   - 确认为硬性错误 → 自动退款,积分退回买家账户
   - 核实为正常号码 → 转入人工审核
5. 退款成功通知买家和卖家
6. 该线索标记为"问题线索"
```

**申诉表单**:
```
申诉类型: [下拉选择]
- 空号/停机
- 错号/非本人
- 其他问题

申诉说明: [文本输入,可选]
```

**处理时效**: 秒级自动处理(依赖运营商接口响应速度)

##### 主观错误申诉(意向不符)

**申诉场景**:
- 客户实际意向与描述严重不符
- 客户已有保单且短期内无购买计划
- 其他影响成交的重大信息不符

**人工仲裁流程**:
```
1. 买家提交申诉申请
2. 上传证据材料:
   - 通话录音(必须)
   - 聊天截图(可选)
   - 其他证明材料
3. 系统通知卖家(48小时内可申诉)
4. 卖家可提交反证材料
5. 管理员介入仲裁
6. 仲裁结果:
   - 买家胜 → 全额退款
   - 卖家胜 → 驳回申诉
   - 部分责任 → 按比例退款
7. 执行退款操作
```

**仲裁评判标准**:
- 意向描述与实际情况偏差程度
- 卖家是否有明显误导行为
- 买家是否恶意申诉
- 通话录音内容分析

**仲裁时效**: 3-5个工作日

### 3.6 积分、提现与税务结算模块

#### 3.6.1 账户体系设计

**充值分账户**:
- 来源: 人民币充值(1:10比例)
- 用途: 仅限购买线索
- 不可提现
- 不可转赠

**收益分账户**:
- 来源: 卖出线索所得
- 用途: 可申请兑换人民币
- 可提现
- 余额显示

**待结算账户**:
- 来源: 已售出但未过售后期的线索
- 状态: 冻结中
- 解冻条件: 24小时无申诉或申诉处理完成

#### 3.6.2 充值流程

**充值入口**:
- 个人中心-钱包页面
- 余额不足时的引导充值

**充值金额档位**:
- 50元 → 500展业分
- 100元 → 1000展业分
- 200元 → 2000展业分
- 500元 → 5000展业分
- 自定义金额

**支付方式**:
- 微信支付
- 支付宝

**充值流程**:
```
1. 选择充值金额
2. 确认支付方式
3. 调起支付SDK
4. 支付成功回调
5. 增加"充值分"账户余额
6. 生成充值订单记录
```

#### 3.6.3 提现流程

**提现条件**:
- 已完成高级认证(人脸识别)
- 已签署《个人众包服务协议》
- "收益分"余额≥100分(最低提现门槛)
- 无未处理的售后纠纷

**提现步骤**:
```
1. 进入提现页面
2. 系统计算可提现金额
3. 输入提现金额(可自定义)
4. 系统自动计算:
   - 第三方支付通道费: 0.6%
   - 税务合规服务费: 1.5%-3%
   - 实际到账金额
5. 确认提现信息
6. 调用灵活用工平台接口
7. 提交提现申请
8. 状态更新为"提现中"
```

**费用计算示例**:
```
提现金额: 1000展业分 (100元)
支付通道费: 100 × 0.6% = 0.6元
税务服务费: 100 × 2% = 2元
实际到账: 100 - 0.6 - 2 = 97.4元
```

**提现到账**:
- 到账时间: 1-3个工作日
- 到账方式: 银行卡转账
- 到账通知: 站内消息+短信

#### 3.6.4 税务合规处理

**灵活用工对接**:
- 合作平台: 云账户或其他第三方众包平台
- 处理方式: 通过灵活用工平台代缴个人所得税
- 纳税凭证: 可在平台下载电子完税证明

**费用结构**:
- 平台手续费: 0%
- 支付通道费: 0.6%(固定)
- 税务合规服务费: 1.5%-3%(浮动,根据金额和税率)

**税务计算规则**:
- 根据提现金额自动计算应纳税额
- 适用劳务报酬所得税率
- 代扣代缴

### 3.7 平台管理后台

#### 3.7.1 风控看板

**核心指标**:
- 今日交易笔数
- 今日交易金额
- 申诉率
- 退款率
- 异常账号数

**信用分系统**:
- 初始信用分: 100分
- 扣分项:
  - 线索质量问题: -5分/次
  - 申诉成立: -10分/次
  - 恶意行为: -20分/次
- 加分项:
  - 交易完成: +1分/笔
  - 好评: +2分/次
- 信用等级:
  - S(90-100): 优质卖家
  - A(80-89): 良好
  - B(70-79): 一般
  - C(60-69): 风险
  - D(<60): 黑名单

**黑名单管理**:
- 自动加入条件:
  - 信用分<60
  - 申诉率>30%
  - 恶意申诉≥3次
- 人工加入: 管理员标记
- 处理措施:
  - 限制发布线索
  - 限制购买线索
  - 冻结账户

#### 3.7.2 交易审核

**审核工单**:
- 待审核列表
- 审核详情页
- 审核操作(通过/驳回)

**复杂纠纷仲裁**:

**审核记录**:
- 操作日志

#### 3.7.3 财务报表

- 充值金额

## 六、接口需求

### 6.1 第三方接口清单

#### 实名认证接口
- **供应商**: 第三方实名认证服务
- **功能**: 三要素认证、人脸识别
- **调用时机**: 用户注册、高级认证
- **费用**: 按次计费

#### 空号检测接口
- **供应商**: 运营商或第三方检测服务
- **功能**: 手机号状态检测
- **调用时机**: 线索上架时
- **费用**: 按次计费

#### 隐私号/虚拟号接口
- **供应商**: 阿里云/腾讯云隐私号服务
- **功能**: 虚拟号外呼、通话录音
- **调用时机**: 买家拨打客户
- **费用**: 按通话时长计费

#### 支付接口
- **供应商**: 微信支付、支付宝
- **功能**: 充值支付
- **调用时机**: 用户充值
- **费用**: 0.6%手续费

#### 灵活用工接口
- **供应商**: 云账户等众包平台
- **功能**: 个人所得税代缴
- **调用时机**: 用户提现
- **费用**: 1.5%-3%服务费

### 6.2 内部API设计

#### 用户认证相关
- POST /api/auth/register - 用户注册
- POST /api/auth/login - 用户登录
- POST /api/auth/verify - 三要素认证
- POST /api/auth/face-verify - 人脸识别

#### 线索管理相关
- POST /api/leads/create - 创建线索
- GET /api/leads/list - 线索列表
- GET /api/leads/{id} - 线索详情
- DELETE /api/leads/{id} - 删除线索
- POST /api/leads/check - AI校验接口

#### 交易相关
- POST /api/orders/create - 创建订单
- POST /api/orders/pay - 支付订单
- GET /api/orders/list - 订单列表
- GET /api/orders/{id} - 订单详情

#### 售后相关
- POST /api/complaints/create - 创建申诉
- GET /api/complaints/list - 申诉列表
- POST /api/complaints/handle - 处理申诉

#### 账户相关
- GET /api/accounts/balance - 查询余额
- POST /api/accounts/recharge - 充值
- POST /api/accounts/withdraw - 提现
